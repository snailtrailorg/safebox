## 二、NIST SP 800-63B

| 要求 | v2.2 | v2.3 | v2.4 | 注释 |
|------|------|------|------|------|
| 记忆秘密最低长度 | ✅ 12 | ✅ 12+复杂度 | ✅ | |
| 失败尝试锁定阈值 | ✅ | ✅ | ✅ | |
| 自动锁定时间 | ✅ | ✅ | ✅ | |
| 速率限制 | ✅ | ✅ L1+L2+L3 | ✅ L1+L2 | Nginx 防 DDoS 粗筛 + 应用层 L1/L2 Redis 限流，两层并存 |
| 多因子鉴证 | ❌ | ✅ | ✅ | 恢复码路径取消验证码，24h 冷却期提供等攻击窗口防护 |
| 账户恢复鉴证 | ❌ | ✅ | ✅ | 恢复码 + 24h 冷却期 + 可取消，等效于双重鉴证 |
| AAL2 物理鉴证器 | ❌ (超出 scope) | ❌ (超出 scope) | ❌ (超出 scope) | |
| 重新认证周期 | 🟡 20 分钟 | 🟡 20 分钟 | 🟡 20 分钟 | |

**适用子集评分：10 项中满足 8 项 = 80%**（v2.2: 60%，评分不变，实现方式从验证码改为冷却期）

> 说明：恢复码路径使用 24h 冷却期 + 加速通道（验证码） + 冻结回滚（签名链接）三层机制。冷却期给用户反应时间；加速通道为合法用户提供跳过等待的选项；冻结回滚利用 rollback_* 副本：initiate 写正式=新密码+存旧密码副本，freeze 回滚 rollback_*。NIST 要求的"账户恢复鉴证"仍然满足。
